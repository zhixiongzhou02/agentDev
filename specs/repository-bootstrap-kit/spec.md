# 仓库启动包规格

## 状态

`active`

## 目的

把当前仓库中的治理结构提炼为一个可复用能力，使维护者能够快速创建新的 AI Agent 项目基础骨架。

## 问题定义

当前示例仓库已经有一套相对清晰的治理结构，但它只能被人工复制。人工复制会带来几个问题：

- 文档容易漏掉
- 目录结构容易漂移
- 协作者会不自觉回到“先写代码、后补文档”的路径
- 不同项目之间很难保持一致的上下文治理规则

## 目标用户

- 需要启动新 AI Agent 项目的维护者
- 希望复制同一套仓库纪律的团队

## 范围内

- 生成受治理的目录结构
- 生成最小基础文档
- 支持项目级变量替换
- 保留 AI 上下文规则

## 范围外

- 生成业务代码
- 生成多套技术栈代码模板
- 托管远程仓库配置

## 成功标准

- 新仓库在初始化后能直接进入 `PLAN -> Requirement -> Design -> Test -> Implementation` 流程
- 生成出的文档足以让人和 Agent 以相同方式理解仓库结构
