# 测试策略

## 目的

这份文档定义仓库如何进行验证与质量保障。

## 策略

仓库默认采用“可追溯的 TDD”作为质量模型。

目标链路是：

`Requirement -> Design -> Test Plan -> Test Case -> Code -> Verification Result`

## 规则

- 重要行为应绑定到明确的需求来源
- 测试应从预期行为出发，而不是围绕偶然的实现细节编写
- 验证证据应记录在计划、测试用例或实现记录中
- 如果代码变更没有自动化测试，也仍然必须记录验证方式

## 测试层次

一旦进入实现阶段，仓库预期具备以下层次：

- 单元测试，用于验证局部行为
- 集成测试，用于验证模块交互
- 契约测试，用于验证外部接口
- 端到端测试，用于验证关键用户流程

## 基础阶段说明

当前仓库仍处于基础建设阶段，因此这份文档先定义策略，再逐步补齐测试资产。
